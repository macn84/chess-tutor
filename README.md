# Chess Tutor

A localhost chess teaching application that acts as an interactive instructor
rather than a simple engine front-end.  As you play, a side panel explains the
*why* behind candidate moves, shows the main-line opening theory for each
position, and lists the opponent's most likely replies so you can study both
sides of the board.

---

## Features

| Feature | Details |
|---|---|
| **Opening book** | 45 hand-curated positions covering all major openings to depth 8–10, with ECO codes, strategic ideas, and prose explanations |
| **Engine analysis** | Stockfish 17.1, multi-PV (4 lines), time-limited at 1.5 s per query so results arrive in under 2 s |
| **Teaching prose** | Every candidate move gets a one-sentence explanation — from the book when in-book, otherwise generated by a template engine from move properties (capture type, central push, development square, check) |
| **Opponent responses** | After each move the top 3 opponent replies appear with explanations and "Play this line" buttons |
| **Evaluation bar** | Animated centipawn bar capped at ±600 cp; displays mate-in-N as ±M |
| **Move history** | Click any move to replay the game from that point; branching replaces future history |
| **Board controls** | Flip board, New Game |
| **My Games tab** | Fetches your Chess.com games by date range, time control, colour, and result; batch-analyses them with Stockfish to detect blunders/mistakes/inaccuracies; aggregates patterns across the whole set; generates coaching insights via Claude (Haiku) or falls back to engine-only analysis when no API key is configured |

|---|---|---|

## Quick Start

```bash
cd chess-tutor
./run.sh
```

`run.sh` will:

1. Source `.env` if present.
2. Activate the Python virtual environment.
3. `pip install -r requirements.txt` (fast on repeat runs — already satisfied).
4. Start Flask on `http://localhost:5000` in the background.
5. Run `npm install --silent` inside `frontend/`.
6. Start the Vite dev server on `http://localhost:5173`.

Open **http://localhost:5173** in your browser.  Press `Ctrl+C` to stop both servers.

## API Reference

All endpoints are served by Flask on `:5000`.  The Vite proxy forwards
`/api/*` from `:5173` so the frontend uses relative URLs and avoids CORS.

### `GET /api/opening?fen=<fen>`

Fast opening-book lookup — no engine call.

**Response (found)**
```json
{
  "found": true,
  "eco": "B20",
  "opening_name": "Sicilian Defense",
  "variation_name": "",
  "strategic_ideas": ["Black fights for the d4 square asymmetrically."],
  "main_line_moves": ["d4", "Nf3", "Nc3"]
}
```

**Response (not found)**
```json
{ "found": false }
```

---

### `POST /api/analyze`

Stockfish multi-PV analysis enriched with opening-book prose.

**Request body**
```json
{ "fen": "<FEN>", "num_candidates": 4 }
```

**Response**
```json
{
  "candidates": [
    {
      "san": "Nf3",
      "uci": "g1f3",
      "score_cp": 30,
      "score_label": "+0.3",
      "pv_san": ["Nf3", "Nc6", "Bb5"],
      "explanation": "Develops the knight to its most natural square (f3).",
      "label": "Ruy Lopez preparation"
    }
  ],
  "best_move_san": "Nf3",
  "eval_cp": 30
}
```

---

### `POST /api/opponent-responses`

Top engine replies from the opponent's perspective, annotated with teaching prose.

**Request body**
```json
{ "fen": "<FEN after your move>", "num_responses": 3 }
```

**Response**
```json
{
  "responses": [
    {
      "san": "e5",
      "score_cp": -30,
      "explanation": "Stakes a claim in the center.",
      "label": "Open Game",
      "follow_up_idea": "White plays 2.Nf3 attacking e5.",
      "resulting_opening": "King's Pawn Game — Open Game",
      "in_book": true
    }
  ]
}
```

---

### `GET /api/legal-moves?fen=<fen>`

Returns all legal SAN moves for a position.  Provided as a fallback; the
frontend uses chess.js locally and rarely calls this endpoint.

---

### `POST /api/games/fetch`

Fetch and filter a player's games from the Chess.com public API.
Username is read from the `CHESS_COM_USERNAME` environment variable.

**Request body**
```json
{
  "start_date": "2024-01-01",
  "end_date":   "2024-03-31",
  "time_class": "rapid",
  "color":      "white",
  "result":     "loss"
}
```
`time_class` defaults to `"all"` (rapid | blitz | bullet | daily | all).
`color` defaults to `"both"`. `result` defaults to `"all"`.

**Response**
```json
{ "games": [...], "count": 47 }
```

HTTP errors: `400` bad params, `502` Chess.com unreachable.

---

### `POST /api/games/analyze`

Start a background batch analysis job and return a job ID immediately.
Username is read from the `CHESS_COM_USERNAME` environment variable.

**Request body**
```json
{ "games": [...] }
```
Maximum 100 games per request.

**Response**
```json
{ "job_id": "uuid", "total": 47 }
```

---

### `GET /api/games/status/<job_id>`

Poll job progress.

**Response**
```json
{ "status": "running", "progress": 0.42, "analyzed_count": 20, "total": 47 }
```
`status` is `running`, `done`, or `error`.

---

### `GET /api/games/results/<job_id>`

Retrieve completed results (only when `status == "done"`; returns 202 otherwise).

**Response**
```json
{
  "patterns": {
    "total_games": 47,
    "wins": 30,
    "blunders_per_game": 1.4,
    "phase_distribution": { "opening": 35, "middlegame": 48, "endgame": 17 },
    "top_blunders": [...],
    "opening_stats": [...],
    "mistake_move_numbers": [...]
  },
  "insights": {
    "insights": "Finding 1: ...",
    "llm_used": true
  }
}
```

## Opening Book Format

`backend/opening_book/book.json` contains a top-level `"entries"` array.
Each entry follows this schema:

```json
{
  "fen_key": "rnbqkbnr/pppppppp/8/8/4P3/8/PPPP1PPP/RNBQKBNR b KQkq -",
  "eco": "B00",
  "opening_name": "King's Pawn Opening",
  "variation_name": "",
  "strategic_ideas": [
    "Controls d5 and e5.",
    "Opens lines for the bishop and queen."
  ],
  "main_line_moves": ["e5", "c5", "e6", "c6"],
  "candidate_explanations": {
    "e5": {
      "label": "Open Game",
      "prose": "Fights for the centre symmetrically. Leads to Ruy Lopez, Italian, Scotch.",
      "follow_up_idea": "White plays 2.Nf3 attacking e5 and preparing Bb5 or Bc4."
    }
  },
  "tags": ["opening", "first-move"]
}
```

**FEN key format**: first four fields only (pieces, side, castling, en-passant).
This strips the half-move clock and full-move number so transpositions — the
same position reached by different move orders — always map to the same entry.

**Openings covered (45 entries)**

| Family | Lines included |
|---|---|
| Open Games (1.e4 e5) | Ruy Lopez (Morphy, Berlin, Closed, Chigorin/Breyer/Zaitsev), Italian (Giuoco Piano, Evans Gambit), Scotch, King's Gambit |
| Semi-open (1.e4) | Sicilian (Open, Najdorf, Dragon, Kan/Taimanov, Alapin), French (Advance, Tarrasch, Winawer), Caro-Kann (Classical, Advance) |
| Closed (1.d4) | Queen's Gambit Accepted/Declined, Nimzo-Indian, King's Indian, Grünfeld, Slav |
| Others | English, London System, King's Indian Attack, Pirc, Vienna, Petrov, Philidor |

## Key Implementation Notes

### Engine Locking

`engine_manager.py` uses a `threading.Lock` **only** for the double-checked
initialisation of the singleton engine.  The `analyse()` call is **never**
wrapped in a lock.  `chess.engine.SimpleEngine` is internally thread-safe;
holding an external lock during `analyse()` deadlocks the asyncio event loop
it uses internally.

### Time vs. Depth Limits

`chess.engine.Limit(depth=18)` with `multipv=4` takes >30 s on this hardware.
All analysis uses `Limit(time=1.5)` which completes in exactly 1.5 seconds
regardless of position complexity.

### react-chessboard v5 API

Version 5 replaced flat props with a single `options={}` object.
`onPieceDrop` no longer receives a plain string for the piece — it receives a
`DraggingPieceDataType` object where `piece.pieceType` is the piece code
(e.g. `"wP"`).

Version 5 also requires **React 19** — it calls `React.use()` internally, which
did not exist in React 18.  Running it with React 18 causes a silent render
crash (blank page with no visible error in production).
